## P0 Critical Fixes (revenue-blocking)

### 1. CORS: checkout + chat blocked on live Lovable domain
**Files:** `supabase/functions/create-checkout/index.ts`, `supabase/functions/chat/index.ts`

## P1 Bug Fixes (wrong behaviour)

### 4. AdminDashboard revenue always showed $0
**File:** `src/pages/admin/AdminDashboard.tsx`
- Fixed field name: `o.total_nzd` → `o.total` (DB schema has `orders.total`, not `total_nzd`)
- Fixed field name in low-stock query: `stock_quantity` → `stock`
- Fixed field name in recent orders table: `customer_name` → `shipping_name`, `customer_email` → `shipping_email`
- Fixed monthly chart: was aggregating only the last 10 orders. Now uses a dedicated `monthlyRes` query spanning the entire current month with no row limit
- Added `Tooltip formatter` to show NZ$ prefix on chart hover

### 5. AdminOrders showed wrong customer info and $0 totals
**File:** `src/pages/admin/AdminOrders.tsx`
- Fixed `customer_name` → `shipping_name`, `customer_email` → `shipping_email`
- Fixed `selected.total_nzd` → `selected.total`

### 6. Compare page prices disagreed with actual checkout prices
**File:** `src/pages/Compare.tsx`
- Removed hardcoded price strings (`NZD$449起`, `NZD$649起`, `NZD$1,099起`)
- Added `useProducts()` hook import
- Added `getLivePrice(tierId)` helper that reads real prices from the DB in the user's current locale/currency
- Price display in both desktop table and mobile cards now shows live DB values

### 7. Password minimum too weak for a payments site
**Files:** `src/pages/Register.tsx`, `src/pages/ResetPassword.tsx`
- Changed `password.length < 6` → `password.length < 8` in both files
- Updated error messages accordingly
